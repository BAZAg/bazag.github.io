1. Пользователь заходит на пустую страничку.
2. Страничка подтягивает скрипт data.js, который находится на том же домене, что и сайт
3. data.js подключает новый скрипт в head temp.js, данный скрипт может находиться на любом другом домене
4. Скрипт temp.js изначально считывает данные (блоки head, body, main, lang), а также строку адреса и параметры.
5. Скрипт содержит в себе список ID категорий.
6. При каждом заходе - скрипт выбирает случайную категорию.
7. При каждом заходе скрипт определяет актуальную языковую версию
8. При каждом заходе скрипт определяет есть ли слово bot в UserAgent
9. В этом скрипте указывается ключик к TGS
10 Изначально, прежде чем что-либо произойдет - устанавливается блок загрузки странички( анимация)
11 Дальше, если в url создержится id - будет произведена либо переадресация,
		речь идёт о прямой отправке пользователя по партнерской ссылке, но как вариант можно использавать кейтаро, подставив ключик потока
	либо отрисовка материала на страничке (в зависимости от того, зашел бот или человек)
	Отрисовка материала производится получением данных для фукнции MyCard с внешнего источника, в идеале используя ID и LANG
	В качестве заглушки залит файл it.js, который содержит пример данных

12 Если ИД не определен - значит считаем что мы зашли на главную - в этом случае выведём материалы случайной категории
	в качестве заглушки (примера данных) используется файл cats.js, в реальных условиях данные должны получаться по ID категории и LANG.
	возможно, после первого получения данных, в базе должна создаваться пометка о том, для какого сайта были выделены данные материалы, чтобы копия материалов не уходила на несколько сайтов.

13 Или же определим, что мы попали на страничку карты сайта - значит отрисуем html карту или переадресуем пользователя на xml (задача отправить туда бота)
	В случае с html подключается файлик отталкиваясь от домена (язык, cats не важны)
	В случае с xml - каждый день возвращаться должна другая карта
	Возможно, в базе данных должны вестись какая-то статистика, если страничка попала в индекс - убирать её с карты сайта

14 Для отрисовки карты сайта создадим новый скрипт, который подгрузится с определенной странички ( другого сайта )
	Для примера, которые будут подгружаться есть в sitemap.js
15. Возможно, стоит использовать реферер, для определения откуда произошла загрузка скрипта...
16. Если до этого момента мы ничего не предприняли - считаем что странички нет и выводим 404

Возможно, есть смысл внедрить какую-то пагинацию по страничкам
Возможно стоит внедрить хлебные крошки
Возможно стоит отрисовать рейтинг
Возможно стоит внедрить комментарии (хотя бы вывод отзывов)
Возможно стоит внедрить переход по категориям (а значит и подтягивать имена категорий откуда-то)
Возможно нужно добавить favicon

Функция Loading просто устанавливает в тег main определенный блок.
Функция Stop404 отображает текст  404 Not Found и указывает что нельзя индексировать эту страничку
Функция CheckBot определяет есть ли определенные слова в юзерагенте, и если есть - возвращает true
Функция GetLang(itemLang) на вход берет 1 элемент массива. Ищет, есть ли в домене этот элемент - и относительно этого устанавливает язык сайта.
Функция version выводит в консоль версию скрипта (в процессе разработки не понятно, обновился уже скрипт или нет)
Функция sitemap(arrayLinks) занимается отрисовкой данных json в виде таблички, формирует html карту сайта, прилетает с внешнего источника
Функция SearchKeyFromLocation(paramInput) ищет параметр и значение в url
Функция MyCard(myJsonData) занимается отрисовкой одной странички, прилетает с внешнего источника
Функция MyCat(myJsonData) занимается отрисовкой списка блоков, прилетат с внешнего источника
Функция MyDraw(myJsonDataItem) рисует один элемент, используется функцией MyCat

Файлик index.html Точка входа на сайт
Файлик 404.html - своя страничка об ошибке
Файлик style.css содержит базовые стили
Файлик sitemap.xsl содержит разметку одной карты сайта
Файлик sitemap-index.xsl содержит разметку странички с множеством карт сайта
Файлик robots.txt содержит параметры индексирования, возможно стоит туда вставить ссылку на карту сайта
Файлик sitemap-all.xml содержит разметку всех карт сайта
Файлик sitemap-19.xml содержит разметку одной карты сайта (около 5000 ссылок)
